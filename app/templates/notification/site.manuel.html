Notification System

This documentation explains how to configure groups, members, trigger types, templates, rules, events, and working hours.

1. Navigate to Notifications

Open:

./notification/

2. Create Groups, Members and Trigger Types

Create Groups

Navigate to:

../notification/members → Create Group

Create or Import Members

A CSV template can be downloaded.

Navigate to:

Create Member
Fields
  • Required Hours
    Defines how many working hours the member must complete during the current calendar year. This value must be configured individually for each member.
    Default: 0
  • Active
    If set to No, the member will not receive emails.

Assign Members to Groups

../notification/members/<id> → Add Group

Create Trigger Types

../notification/trigger-types → Create Trigger Type
Do not modify the codes: BIRTHDAY, MEMBER_ANNIVERSARY, WORKING_HOURS_MONTHLY

Trigger Types define events that Rules can react to later.

BIRTHDAY

Triggered on a member's birthday.

This trigger is only executed exactly at the configured number of days before the event. If the trigger is not processed within that exact 24-hour window, no notification will be sent later.
MEMBER_ANNIVERSARY

Triggered on a member's join date anniversary.

This trigger is only executed exactly at the configured number of days before the event. If the trigger is not processed within that exact 24-hour window, no notification will be sent later.
WORKING_HOURS_MONTHLY

Sends the member's completed working hours monthly. Delivery occurs on the 1st day of each month.

This trigger only executes on the scheduled day. If processing fails on that day, the notification will not be sent later.
Other Trigger Types may still trigger later, as long as the related event has not already ended.
You can bypass the 24h processing window for BIRTHDAY, MEMBER_ANNIVERSARY and WORKING_HOURS_MONTHLY triggers by running the python mail-service.py --debug script. But be careful: This may result in duplicate emails if the notification log is cleared too early. There is no 100% gurantee this will work in every situation, but it can be helpful for setup, testing and debugging purposes.

These Trigger Types require both:

  • a Template
  • at least one Rule

3. Create Templates

Navigate to:

../notification/templates → Create Template

Templates are later sent as emails. Placeholders can be used in both the subject and body and are automatically replaced during delivery.

Example 1 — Event Reminder

Subject
Reminder: {{ '{{event.title}}' }} on {{ '{{event.start_at}}' }}
Body
Hello {{ '{{receiver.first_name}}' }},

you have an upcoming kitchen duty.

When: {{ '{{event.start_at}}' }} - {{ '{{event.end_at}}' }}
Where: {{ '{{event.location}}' }}

Example 2 — Membership Anniversary

Subject
REMINDER: 25 Years Membership of {{ '{{member.first_name}}' }} {{ '{{member.last_name}}' }}
Body
Hello {{ '{{receiver.first_name}}' }},

member {{ '{{member.first_name}}' }} {{ '{{member.last_name}}' }}
has their 25-year membership anniversary on
{{ '{{add_years(member.join_date, 25)}}' }}.

4. Create Rules

Rules define when notifications are sent.

Recommended naming format:

  • KitchenDuty_7d
  • KitchenDuty_1d

This allows multiple reminders for the same event.

You can assign:
  • single email addresses
  • or entire groups
Every assigned recipient will receive a notification whenever the Rule is triggered.

Example 1 — Kitchen Duty

Name KitchenDuty_7d
Trigger Type KitchenDuty
Template KitchenDuty
Days Before 7
Send Time 08:00
Active Yes

Example 2 — 25-Year Anniversary

Name 25_MEMBERSHIP_7d
Trigger Type MEMBER_ANNIVERSARY
Years 25
Template 25_Years_Membership
Days Before 7
Active Yes
If a Rule is active, all assigned recipients will receive a notification whenever the Rule is triggered.

5. Create Events

Events can be created in three different ways:

  1. Import an .ics file (template available)
  2. Import a .csv or .tsv file (template available)
  3. Create events manually using the web form

Example — Kitchen Duty

Title Kitchen Duty
Trigger Type KitchenDuty
Start At 13.12.2026 19:00:00
End At Optional
Location Clubhouse or address
Recurring No
Supported recurrence:
  • Daily
  • Weekly (weekday selection currently unsupported)
  • Monthly
After creating an event, members can be added as Participants. All Participants receive notifications whenever a matching trigger is executed for that event.
No events are required for: BIRTHDAY, MEMBER_ANNIVERSARY, or WORKING_HOURS_MONTHLY.

6. Trigger Behavior

  • Triggers are only executed while the related event has not yet ended.
  • No notifications are sent for past events.
  • Duplicate emails are automatically prevented.
  • Exception: If the notification log is deleted or cleared too early, duplicate emails may be sent again.
